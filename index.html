Добро пожаловать в GitHub Pages.

Это автоматический генератор страницы это самый простой способ для создания красивых страниц для всех ваших проектов. Автор содержания страницы здесь с помощью GitHub Ароматизированные Markdown, выберите шаблон созданный дизайнером, и опубликовать. После того, как страница создана, вы можете проверить новые отрасли:

$ cd your_repo_root/repo_name
$ git fetch origin
$ git checkout gh-pages

Если вы используете GitHub для Mac, просто синхронизировать хранилища, и вы увидите новую ветку.

Шаблоны Designer

Мы созданный некоторые красивые шаблоны, которые можно использовать. Идем дальше и продолжать макетов для просмотра через них. Вы можете легко вернуться к редактировать свои страницы перед публикацией. После публикации вашей страницы, вы можете вернуться на страницу генератор и переключиться на другую тему. Содержания страницы будут сохранены, если он остался уценки формате.

Вместо Drive Stick?

Если вы предпочитаете не использовать автоматический генератор, нажмите филиала именем GH-страниц для вашего хранилища, чтобы создать страницу вручную. В дополнение к поддержке регулярных HTML содержание, GitHub страницы поддерживают Джекила, простой, блог знать статический генератор сайте написано нашими собственными Tom Preston-Werner. Jekyll позволяет легко создавать веб-узле верхние и нижние колонтитулы, без необходимости копировать их через каждую страницу. Он также предлагает интеллектуальную поддержку блогов и других передовых шаблонов функций.

Авторы и участники

Вы можете отметить @ GitHub имя пользователя, чтобы создать ссылку на свой профиль. В результате `` элементом будет ссылка на GitHub профиль вкладчика. Например: In 2007, Chris Wanstrath (@defunkt), PJ Hyett (@pjhyett), and Tom Preston-Werner (@mojombo) founded GitHub.
